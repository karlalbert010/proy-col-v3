ATENCIÓN CODEX:
Este sistema NO debe diseñarse como un CRUD genérico de base de datos.
Reglas obligatorias:

NO mostrar tablas completas innecesariamente.
SIEMPRE priorizar vistas operativas (lo que el usuario necesita hacer).
El sistema está orientado a:
operadores administrativos
profesores
La navegación debe estar optimizada por frecuencia de uso.
Evitar pantallas con exceso de datos (overfetching visual).
Cada vista debe tener un propósito claro (acción).
Los CRUD completos solo deben existir en módulos de gestión, no en operación diaria.

El diseño debe ser tipo sistema profesional (ERP educativo), no un panel técnico.
