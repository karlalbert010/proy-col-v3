==============================================================
FRONTEND - ARBOL GENERAL DE NAVEGACION Y FLUJO
Sistema Colegio - PC / Profesor / Administrativo / Directivo
==============================================================

OBJETIVO GENERAL
- Que cada item se ingrese mediante formularios CRUD.
- Reutilizar la mayor cantidad posible de código.
- Dar acceso rápido desde menú principal a los items de mayor uso.
- Adaptar el menú según el rol del usuario.
- Permitir flujo fluido para operador PC, profesor, directivo y admin.

2. PRINCIPIO CLAVE DE REUTILIZACION
==============================================================

Todo item del sistema debe derivar de un patrón base:

Entidad
 ├── Listar
 ├── Crear
 ├── Editar
 ├── Ver detalle
 ├── Activar / Desactivar
 └── Eliminar (si la regla de negocio lo permite)

Plantilla estándar de módulo CRUD:
├── /modulo/list
├── /modulo/new
├── /modulo/:id
├── /modulo/:id/edit
└── /modulo/:id/history   (si aplica logs)

Componentes reutilizables:
- Tabla reutilizable con columnas configurables
- Formulario reutilizable por schema
- Validaciones reutilizables
- Botonera CRUD reutilizable
- Selector de curso / año / período reutilizable
- Selector de docente / alumno / materia reutilizable
- Modal de confirmación reutilizable
- Badges de estado reutilizables

8. DISEÑO DE PANTALLAS CRUD REUTILIZABLES
==============================================================

Cada pantalla CRUD debe tener:

CABECERA
├── Título módulo
├── Breadcrumb
├── Acciones rápidas
└── Estado actual

CUERPO
├── Barra de filtros
├── Tabla o grilla
├── Paginación
├── Acciones por fila
└── Panel lateral opcional de detalle

FORMULARIO
├── Campos principales
├── Validación inline
├── Botones Guardar / Guardar y seguir / Cancelar
└── Mensajes de error claros

PIE
├── Estado de guardado
├── Usuario que modificó
└── Fecha de última modificación

==============================================================
11. CRITERIO DE FLUIDEZ DE INTERACCION
==============================================================

Para que la interacción sea fluida:
- Menú siempre visible en PC.
- Accesos rápidos arriba del dashboard.
- Búsqueda global de alumno / docente / curso.
- Menos clics para tareas frecuentes.
- Formularios cortos y claros.
- Grillas rápidas en carga masiva.
- Persistencia de filtros usados.
- Recordar último curso/período usado por profesor.
- Botones contextuales según rol.
- Feedback inmediato al guardar.

==============================================================
12. IDEA CENTRAL DEL FRONTEND
==============================================================

EL FRONTEND NO DEBE SER UNA COLECCION DE PANTALLAS AISLADAS.

DEBE SER:
- modular,
- reutilizable,
- orientado a roles,
- orientado a frecuencia de uso,
- con CRUD base para todas las entidades,
- con grillas rápidas para operaciones masivas,
- y con navegación simple desde un menú principal inteligente.
